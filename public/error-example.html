‚úÖ Example 4: Valid POST Request

Create a new user with complete, valid JSON:

  "email": "jane.smith@example.com",
  "age": 28
}

‚úÖ Example 5: Valid PUT Request

  "email": "alice.updated@example.com",
  "age": 31
}

Expected Result: Status 200 - Resource updated successfully

‚ùå Example 6: Wrong Data Types

  "email": "john@example.com",
  "age": "thirty"    // ‚ùå Age should be a number, not a string
}

Note: json-server may still accept this, but real APIs would validate types

‚úÖ Example 7: Valid Order Creation

  "quantity": 3,
  "status": "pending"

üéØ Testing Tips

  • JSON Syntax: Always use double quotes for property names and string values
  • No Trailing Commas: Don't add commas after the last property
  • Check IDs: Make sure resource IDs exist before trying to update or delete
  • Watch Status Codes:
    • 200 = Success (OK)
    • 201 = Created
    • 400 = Bad Request (invalid data)
    • 404 = Not Found
    • 500 = Server Error
